## 实现 AST Visitor 模式

生成了设计文档 `AST_VISITOR_DESIGN.md`，然后实现了完整的 Visitor/Transformer 框架。

### 架构概览

```
                ASTVisitor[R]（只读遍历，泛型返回值）
                    ├── TableExtractor      → List[String]  提取表名
                    ├── ColumnExtractor     → List[String]  提取列名
                    └── SQLPrettyPrinter    → String         SQL 格式化

                ASTTransformer（变换遍历，返回新 AST）
                    ├── TableRenamer        表名重命名
                    └── ColumnRenamer       列名重命名
```

### 新建文件
1. **ASTVisitor.scala**（新建）— 包含所有 Visitor/Transformer 代码
   - `ASTVisitor[R]` trait — 泛型只读遍历器（Monoid: zero + combine）
     - 默认深度优先递归遍历，子类只需 override 关注的节点方法
     - 覆盖所有 AST 节点：7 种语句 + 4 种列 + 3 种表引用 + 19 种表达式
   - `ASTTransformer` trait — AST 变换器
     - 默认递归变换重建所有子节点
     - 原子钩子：`transformTableName()` / `transformColumnName()` / `transformQualifier()`
   - `TableExtractor` — 提取所有引用的表名（FROM/JOIN/INSERT/UPDATE/DELETE/CREATE/DROP/子查询）
   - `ColumnExtractor` — 提取所有引用的列名（SELECT/WHERE/INSERT/UPDATE/GROUP BY 等）
   - `SQLPrettyPrinter` — AST → 格式化 SQL 字符串（关键字大写 + 缩进对齐 + 子句独立行）
   - `TableRenamer(mapping)` — 按映射表重命名所有表名
   - `ColumnRenamer(mapping)` — 按映射表重命名所有列名

### 修改的文件
2. **MySQLParser.scala** — main 方法新增 Visitor 模式演示（5 个场景：表提取、列提取、SQL 格式化、表重命名、列重命名）

## 实现基于 Visitor 模式的语义分析

将语义分析从手写递归（`SemanticAnalyzer` 对象）重构为基于 `ASTVisitor[R]` 框架的 Visitor 管道模式。

### 架构概览

```
SemanticVisitorPipeline（组合多个 Visitor 的管道）
    ├── TableExistenceVisitor      — 表存在性检查
    ├── ColumnExistenceVisitor     — 列存在性检查（含 JOIN ON 条件）
    ├── GroupByConsistencyVisitor   — GROUP BY 一致性检查
    ├── AliasUniquenessVisitor     — 列别名唯一性检查
    ├── HavingConsistencyVisitor   — HAVING 子句合法性检查
    ├── InsertValidationVisitor    — INSERT 列数匹配检查
    ├── UnionColumnCountVisitor    — UNION 列数一致性检查
    └── DDLValidationVisitor       — CREATE/DROP TABLE 检查
```

### 核心设计
- **SemanticBaseVisitor** — 所有语义 Visitor 的基类，`R = List[SemanticError]`，内置 `error()` / `warning()` 便捷方法
- **SemanticVisitorPipeline** — 组合管道，对 AST 执行一次遍历收集所有语义错误
  - `withOnly(classes)` — 创建只包含指定 Visitor 的子管道
  - `without(classes)` — 创建排除指定 Visitor 的子管道
- **ScopeBuilder** — 作用域构建辅助对象（从 FROM/JOIN 提取可用表和列）
- **ExpressionHelper** — 表达式辅助工具（提取标识符、判断聚合函数）

### 新建文件
1. **SemanticVisitor.scala**（新建）— 包含所有 Visitor 语义分析代码

### 修改的文件

  - 遵循开放-封闭原则
